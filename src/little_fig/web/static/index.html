📖 Little Fig Documentation

v0.6 · FigQuant + FigKernel + FigPipeline + Ember

🚀 Quick Start

  1. Enter a HF model ID in the Chat sidebar (e.g. Qwen/Qwen2.5-0.5B-Instruct)
  2. Click Load and wait for the status badge to turn green
  3. Type a message and press Enter

🔧 Engine Components

FigQuant

Adaptive codebook INT4 quantization. NF4 base refined via weighted k-means. 9.7% less MSE than standard INT4.

FigKernel

torch.compile fused ops: RMSNorm (2.95× faster), SwiGLU, chunked cross-entropy, fused Linear+LoRA.

FigPipeline

Async GPU-CPU training. Optimizer states on CPU, compute on GPU. Enables training models larger than VRAM.

Ember Memory

Cognitive memory token injection. Train models to store, recall, consolidate, and forget — embedded in weights.

🏋️ Training Tiers

Tier 1: Streaming LoRA — INT4 base + LoRA adapters. ~400MB for 1.1B. Good quality, fast.

Tier 2: LISA — Rotating layer unfreezing. ~900MB for 1.1B. 10-35% better than LoRA.

Tier 3: MeZO — Zeroth-order (no backward pass). ~600MB for 1.1B. Acceptable, slow.

Tier 4: LOMO — Fused backward+update. ~800MB for 1.1B. Best quality, fast.
