Little Fig

Load a model from the sidebar, then start chatting. Supports text, images, and file attachments.

🏋️ Training Ground

Fine-tune models with Fig Engine. Four training tiers auto-selected based on your RAM.

Tier 1: Streaming LoRA

INT4 base + FP32 LoRA adapters. ~400MB for 1.1B. Fast.

Tier 2: LISA

Rotating layer unfreezing. 10-35% better than LoRA.

Tier 3: MeZO

Zero-order optimization. No backward pass needed.

Tier 4: LOMO

Fused gradient + update. Full fine-tune quality.

📖 Little Fig Documentation

Quick Start

  1. Load a model — Type a HuggingFace model ID (e.g. Qwen/Qwen2.5-0.5B-Instruct) or a local path to a .gguf file in the sidebar, then click Load.
  2. Chat — Type a message and press Enter. Responses stream in real-time via WebSocket.
  3. Attach files — Click the 📎 icon to attach images, PDFs, code files, or audio.

Model Loading

Little Fig's universal GGUF loader auto-detects any architecture. It tries:

  1. transformers built-in (22+ architectures)
  2. gguf-py TensorNameMap fallback (122+ architectures)
  3. Auto-upgrades transformers if architecture is too new

You can also use the 📁 Browse button to navigate your local filesystem.

Fig Engine

The core engine uses INT4 streaming quantization (7.4× compression) with adaptive codebook fitting (FigQuant) and torch.compile fused operations (FigKernel).

ComponentWhat it does
FigQuantAdaptive codebook INT4 — 9.7% less error than standard INT4
FigKernelFused ops via torch.compile — 2.95× faster RMSNorm
FigPipelineAsync GPU-CPU training — optimizer states on CPU
FigLinearINT4 base + LoRA adapters, fast/low-RAM modes

Training (Python API)

from little_fig.engine import FigModel, FigTrainer, FigTrainingConfig

model = FigModel.from_pretrained("TinyLlama/TinyLlama-1.1B-Chat-v1.0", lora_r=16)
config = FigTrainingConfig(num_epochs=3, learning_rate=2e-4)
trainer = FigTrainer(model, config)
trainer.load_dataset("tatsu-lab/alpaca")
trainer.train()

Troubleshooting

Model fails to load? Run pip install --upgrade transformers gguf and restart.

Out of memory? Use a smaller model (GPT-2, Qwen 0.5B) or free up RAM.

Gemma 4 not loading? Your transformers version needs updating: pip install git+https://github.com/huggingface/transformers.git
